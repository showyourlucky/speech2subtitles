# 系统音频捕获使用指��

## 概述

本指南介绍如何使用 Speech2Subtitles 系统的音频捕获功能来转录播放中的视频、音乐或其他系统音频。

## 功能特点

- ✅ **实时捕获**: 捕获Windows系统播放的任何音频
- ✅ **高质量转录**: 基于sherpa-onnx + sense-voice模型
- ✅ **低延迟**: 实时转录，延迟小于2秒
- ✅ **双后端支持**: 优先使用soundcard（无杂音），自动回退到PyAudio
- ✅ **智能选择**: 自动选择最佳音频捕获后端
- ✅ **高音质**: 直接从扬声器获取环回音频，避免立体声混音杂音

## 前置条件

### 1. 硬件要求
- Windows 10/11 操作系统
- 支持音频输出的扬声器或耳机

### 2. 软件要求
- Python 3.8+
- PyAudio 库（可选，作为回退方案）
- soundcard 库（推荐，提供更高质量的音频捕获）
- 已安装的项目依赖

### 3. 音频设备设置

**推荐设置（使用soundcard后端）**:
- 确保系统有默认扬声器设备
- 无需额外配置，自动适配系统音频设备

**备用设置（使用PyAudio立体声混音后端）**:

**Windows 10/11 设置步骤**:

1. **打开声音设置**
   - 右键任务栏右下角的音量图标
   - 选择"声音设置"

2. **进入声音控制面板**
   - 在声音设置页面中找到"声音控制面板"链接
   - 点击进入

3. **启用立体声混音**
   - 切换到"录制"选项卡
   - 右键空白处，勾选"显示禁用的设备"
   - 找到"立体声混音"(Stereo Mix)
   - 右键点击，选择"启用"

4. **设置为默认设备**
   - 右键点击"立体声混音"
   - 选择"设置为默认设备"

5. **验证设置**
   - 立体声混音应该显示为默认设备（带绿勾）
   - 如果没有看到"立体声混音"，可能需要更新声卡驱动

## 使用方法

### 方法1: 测试系统音频捕获

首先运行测试工具验证功能是否正常：

```bash
python test_system_audio_simple.py
```

**预期输出**:
```

   [SUCCESS] 找到系统音频设备: 立体声混音 (Realtek HD Audio Stereo input)

   [SUCCESS] 设备验证成功

4. 测试音频捕获 (10秒):
   请播放一些音频内容...
   [SUCCESS] 音频捕获测试成功

   捕获速率: 46.9 chunks/s

[SUCCESS] 系统音频捕获功能正常!
```

### 方法2: 完整媒体转录

使用完整的转录功能进行实时转录：

```bash
python main.py \
    --model-path models/sherpa-onnx-sense-voice-zh-en-ja-ko-yue-2024-07-17/model.onnx \
    --input-source system
```

**可选参数**:
```bash
# 禁用GPU加速（如果遇到CUDA问题）
python main.py --model-path <模型路径> --input-source system --no-gpu

# 调整VAD阈值（0.1-0.9，默认0.5）
python main.py --model-path <模型路径> --input-source system --vad-threshold 0.7

# 指定音频设备ID（如果有多设备）
python main.py --model-path <模型路径> --input-source system --device-id 24
```

### 方法3: 调试模式

如果遇到问题，可以使用调试模式获取更多信息：

```bash
python main.py --model-path <模型路径> --input-source system --log-level DEBUG
```

## 故障排除

### 常见问题及解决方案

#### 问题1: "未找到系统音频设备"

**解决方案**:
1. 检查立体声混音是否已启用
2. 确认立体声混音已设为默认设备
3. 重启音频服务：
   ```
   services.msc → Windows Audio → 重启
   ```

#### 问题2: 设备验证失败

**解决方案**:
1. 更新声卡驱动程序
2. 检查Windows音频服务是否运行
3. 尝试不同的音频格式设置

#### 问题3: 转录延迟过高

**解决方案**:
1. 使用GPU加速（如果可用）
2. 调整VAD阈值参数
3. 检查系统性能（CPU/内存使用率）

#### 问题4: 转录准确率低

**解决方案**:
1. 确保音频质量良好（无噪音）
2. 调整VAD阈值（0.1-0.7之间）
3. 使用高质量音频源

#### 问题5: PyAudio安装失败

**解决方案**:
```bash
# 方法1: 使用pipwin
pip install pipwin
pipwin install pyaudio

# 方法2: 手动安装wheel文件
pip install PyAudio-0.2.11-cp310-cp310-win_amd64.whl
```

## 性能优化建议

### 1. 系统优化
- 关闭不必要的后台程序
- 确保有足够的内存（推荐8GB+）

- 根据需要调整VAD阈值
- 选择合适的模型文件大小
