# Speech2Subtitles GUI 使用指南

## 概述

Speech2Subtitles现已提供基于PySide6的图形用户界面（GUI），提供更友好的用户体验。

## 安装依赖

### 1. 激活虚拟环境

```bash
# Windows
.venv\Scripts\activate

# Linux/macOS
source .venv/bin/activate
```

### 2. 安装PySide6

```bash
# 使用uv（推荐）
uv pip install PySide6

# 或使用pip
pip install PySide6
```

## 启动GUI

### 方式1: 直接运行GUI启动程序

```bash
python gui_main.py
```

### 方式2: 使用主程序（无参数启动）

未来版本将支持通过`main.py`无参数启动GUI模式。

## GUI功能

### 主界面布局

GUI分为左右两个面板：

#### 左侧面板（控制区域）
1. **音频源选择器**
   - 麦克风：使用系统默认麦克风
   - 系统音频：捕获系统音频输出
   - 音频/视频文件：选择本地媒体文件

2. **转录控制面板**
   - 开始转录按钮
   - 暂停按钮（计划功能）
   - 停止按钮
   - 状态指示器
   - 转录时长显示

3. **实时状态监控**
   - 转录状态
   - 音频源信息
   - 模型信息
   - GPU状态
   - 音频电平（实时）
   - 处理延迟

#### 右侧面板（显示区域）
- **转录结果显示**
  - 带时间戳的转录文本
  - 自动滚动
  - 清空和复制功能
  - 字数统计

### 菜单栏
- **文件菜单**
  - 退出 (Ctrl+Q)
- **帮助菜单**
  - 关于

## 使用流程

### 实时转录（麦克风）

1. 启动GUI: `python gui_main.py`
2. 在音频源选择器中选择"麦克风"
3. 点击"开始转录"按钮
4. 对着麦克风说话
5. 转录结果实时显示在右侧面板
6. 点击"停止"按钮结束转录

### 实时转录（系统音频）

1. 启动GUI
2. 选择"系统音频"
3. 点击"开始转录"
4. 播放音频/视频
5. 转录结果实时显示
6. 停止转录

**注意（Windows）**: 使用系统音频前需启用"立体声混音"设备。

### 文件转录

1. 启动GUI
2. 选择"音频/视频文件"
3. 点击"浏览..."选择文件
4. 点击"开始转录"
5. 等待处理完成
6. 查看结果

## 配置

### 配置文件位置与格式

GUI 配置默认保存在项目根目录：

```text
config/gui_config.json
```

配置文件采用 `schema v2` 包裹格式：

```json
{
  "version": "2.0",
  "last_modified": "2026-04-07T12:00:00",
  "config": {
    "runtime": {},
    "audio": {},
    "vad": {},
    "output": {},
    "subtitle": {}
  }
}
```

### GUI 配置加载与保存行为

- 启动 GUI 时，配置通过 `ConfigLoader` 加载。
- 合并优先级与 CLI 保持一致：`CLI(仅显式参数) > ENV > 文件 > 默认`。
- `config/gui_config.json` 负责保存并提供全部常用参数（模型、VAD、音频、输出、字幕显示）。
- 未显式传入的 CLI 参数不会覆盖配置文件，系统会继续使用 `gui_config.json` 中当前激活方案的值。
- 在设置对话框点击“保存”后，会写回 `config/gui_config.json`。
- 支持“导入配置 / 导出配置”。
- 导入旧版本配置时会自动迁移到 v2。

### 兼容字段映射（GUI -> v2）

为了兼容旧控件键名，`ConfigBridge.update_config()` 会自动做映射：

- 平铺键示例：`model_path -> runtime.model_path`
- 平铺键示例：`output_format -> output.format`
- 嵌套键示例：`subtitle_display.enabled -> subtitle.display.enabled`

### 模型与 GPU

- 模型路径支持在设置对话框中直接选择并验证（`.onnx` / `.bin`）。
- GUI 会自动检测 GPU 可用性，并允许通过配置开关启用/禁用。

### 资源管理优化 ⚡

**TranscriptionEngineManager** (v0.1.1新增)

智能管理转录模型的生命周期，显著提升用户体验：

**核心特性**：
- ✅ **智能复用**: 配置未变化时复用已加载模型（避免~2GB重新加载）
- ✅ **自动检测**: 配置变化时自动重新加载（model_path, use_gpu, sample_rate）
- ✅ **显式控制**: 仅在关闭窗口时释放资源，停止按钮不释放
- ✅ **性能提升**: 停止后再启动从1.5-3秒降至<100ms（**15-30x提升**）

**资源释放策略**：
1. **关闭主窗口**: 完全释放所有模型资源
2. **模型配置改变**: 自动释放旧模型，加载新模型
3. **停止转录**: 保持模型加载，支持即时重启

**统计监控**：
```python
# 查看引擎使用统计
stats = TranscriptionEngineManager.get_statistics()
print(f"加载次数: {stats['engine_loads']}")
print(f"复用次数: {stats['engine_reuses']}")
```

### 线程模型

- **主线程**: 运行Qt事件循环，处理UI交互
- **Pipeline线程**: 独立线程运行TranscriptionPipeline，避免阻塞GUI
- **事件通信**: 通过Qt信号槽机制实现线程安全的通信

### 信号槽连接

```
用户操作 -> UI组件信号 -> MainWindow槽函数 -> Pipeline操作
Pipeline事件 -> PipelineBridge信号 -> UI组件槽函数 -> UI更新
```

## 故障排除

### 问题1: PySide6导入失败

**错误信息**: `ModuleNotFoundError: No module named 'PySide6'`

**解决方案**:
```bash
uv pip install PySide6
```

### 问题2: 窗口无法显示

**可能原因**:
- 显示驱动问题
- Qt平台插件缺失

**解决方案**:
```bash
# 检查Qt插件
python -c "from PySide6.QtWidgets import QApplication; print(QApplication.platformName())"
```

### 问题3: 音频源无法选择

**可能原因**:
- 麦克风权限未授予
- 音频设备驱动问题

**解决方案**:
- 检查系统麦克风权限设置
- 确保音频设备正常工作

### 问题4: 转录无结果

**可能原因**:
- 模型路径错误
- 音频输入静音
- VAD敏感度过高

**解决方案**:
- 检查模型文件路径
- 确认音频电平有变化
- 查看日志输出
