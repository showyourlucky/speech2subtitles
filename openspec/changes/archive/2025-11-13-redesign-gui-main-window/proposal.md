# Proposal: redesign-gui-main-window

## Background

当前GUI主界面存在以下问题:

1. **信息密度过高**: 左侧面板包含了音频源选择、VAD方案、转录控制、实时状态监控等众多元素,内容过于拥挤
2. **层次不清晰**: 所有控制都堆在左侧单一面板中,缺乏视觉分组和层次感
3. **空间利用不均**: 左侧拥挤而右侧只有转录结果显示,空间分配不合理
4. **交互不够简洁**: 音频源使用单选按钮组,文件模式下文件列表始终占用空间
5. **转录控制冗余**: 控制区域包含过多信息,实际高频使用的只是开始/停止按钮
6. **屏幕适配问题**: 在较小屏幕(1024x768)上无法完整显示所有控制元素

从用户提供的截图可以看到,当前界面存在布局拥挤和空间分配不合理的问题。

**用户反馈** (2025-11-11):
- 希望音频源改成下拉选项,更简洁
- 文件列表仅在选择"文件"时显示,节省空间
- 转录控制简化,统计信息移到状态栏
- 音频电平等高级功能放入可折叠面板

## Proposed Solution

重新设计GUI主界面,采用**上下分区布局**和**按需展开**的交互模式,提升用户体验:

### 核心设计理念 (v2.0)

1. **上下分区**: 顶部控制区+下部结果显示区,视觉更清晰
2. **按需展开**: 文件列表和高级设置仅在需要时显示,减少视觉负担
3. **简化控制**: 只保留核心按钮,统计信息移至状态栏
4. **快速访问**: 高频操作(开始/停止/音频源切换)一目了然

### 关键改进 (v2.0)

1. **音频源下拉框**: 从单选按钮组改为下拉选择,更简洁紧凑
2. **音频电平前置**: 从高级设置移到音频源下方,实时监控更直观
3. **按需展开文件列表**: 仅在选择"文件"时展开文件选择面板,节省空间
4. **简化控制区**: 只保留开始/暂停/停止按钮和高级设置入口
5. **状态栏集成**: 统计信息(转录句数、时长、GPU状态)移至底部状态栏
6. **高级设置简化**: 移除音频电平,模型改为下拉选择,减少二级交互
7. **上下分区**: 控制区和结果区分离,视觉层次更清晰
8. **响应式布局**: 支持窗口大小调整,最小800x600

## Goals

1. **提升易用性**: 让新用户能够快速理解和使用界面
2. **提高效率**: 减少高频操作的点击次数
3. **改善视觉**: 打造现代、专业、美观的界面
4. **增强适配性**: 支持不同屏幕尺寸,避免显示不完整
5. **保持兼容**: 不影响现有功能和配置系统

## Alternatives Considered

### 方案1: 标签页布局
使用标签页将"控制"、"监控"、"设置"分离到不同标签页。

**优点**: 最大限度减少视觉复杂度
**缺点**: 需要切换标签页查看不同信息,不适合实时监控场景

### 方案2: 侧边抽屉式布局
使用可隐藏的侧边抽屉存放高级设置。

**优点**: 更现代的交互方式
**缺点**: 在Windows桌面应用中不够符合用户习惯

### 选择方案: 卡片+折叠面板
既保持信息的可见性,又减少初始复杂度,符合桌面应用习惯。
