# Proposal: 文件选择支持复选框多选删除

## 变更ID
`add-file-checkbox-multiselect`

## 状态
📝 **提案中** (Proposed)

## 概述

为GUI的音频/视频文件选择功能添加复选框支持，实现批量选择和批量删除文件的能力。当前实现仅支持单选删除（点击选中某一项后删除），本变更将增强用户体验，允许用户通过复选框同时选择多个文件进行批量删除操作。

## 动机

### 当前痛点
1. **单选删除效率低**：当用户添加了10+个文件后，如果想删除其中多个文件，需要重复点击"移除选中"按钮多次
2. **操作繁琐**：每次只能删除一个文件，用户体验不佳
3. **缺少批量操作**：没有提供"全选"、"反选"等常见批量操作功能

### 使用场景
- **场景1**：用户批量添加了20个视频文件，发现其中5个格式不对或文件损坏，希望一次性删除这5个文件
- **场景2**：用户添加了多个测试文件，想保留部分正式文件，需要快速删除测试文件
- **场景3**：用户在文件列表中发现重复添加了相同文件，希望批量清理重复项

## 目标

### 核心功能
1. ✅ 在文件列表的每一项前添加复选框
2. ✅ 添加"全选"/"取消全选"按钮
3. ✅ 修改"移除选中"按钮逻辑，支持删除所有勾选的文件
4. ✅ 保持现有的单项选择高亮功能（用于显示详情）
5. ✅ 更新按钮状态逻辑（根据复选框状态启用/禁用）

### 非目标
- ❌ 不涉及文件内容的验证或预览
- ❌ 不改变文件转录的核心流程
- ❌ 不添加文件拖拽排序功能

## 实现策略

### 技术方案
使用PySide6的`QListWidget`配合自定义`QListWidgetItem`，为每个文件项添加复选框：

**方案选择**：
- **方案A（推荐）**：使用`Qt.ItemIsUserCheckable`标志，为每个`QListWidgetItem`添加内置复选框
  - 优点：原生支持，代码简洁，性能好
  - 缺点：样式定制能力有限

- **方案B**：自定义`QWidget`包含`QCheckBox` + `QLabel`，通过`setItemWidget`设置
  - 优点：样式完全可控
  - 缺点：代码复杂度高，性能略差

**选择方案A**，理由：符合KISS原则，满足功能需求且实现简单。

## 风险评估

### 技术风险
- **风险等级**：🟢 低
- **主要风险**：
  1. 复选框状态管理可能与单项选择高亮冲突
     - **缓解措施**：复选框和高亮选择是独立的Qt机制，测试验证即可

  2. 大量文件（100+）时复选框操作可能卡顿
     - **缓解措施**：当前设计支持的文件数量通常<50，性能风险极低

### 兼容性风险
- **风险等级**：🟢 无
- 纯UI增强，不改变数据结构和API
