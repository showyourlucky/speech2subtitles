# Proposal: redesign-gui-main-window

## Background

当前GUI主界面存在以下问题:

1. **信息密度过高**: 左侧面板包含了音频源选择、VAD方案、转录控制、实时状态监控等众多元素,内容过于拥挤
2. **层次不清晰**: 所有控制都堆在左侧单一面板中,缺乏视觉分组和层次感
3. **空间利用不均**: 左侧拥挤而右侧只有转录结果显示,空间分配不合理
4. **缺乏现代感**: 整体布局较为传统,缺少现代应用的视觉吸引力和易用性
5. **屏幕适配问题**: 在较小屏幕上无法完整显示所有控制元素

从用户提供的截图可以看到,当前界面在1024x768等分辨率下会出现显示不完整的问题。

## Proposed Solution

重新设计GUI主界面,采用**现代化的卡片式布局**和**响应式设计**,提升用户体验:

### 核心设计理念

1. **卡片式布局**: 使用视觉卡片组织不同功能区域,提供清晰的视觉分组
2. **扁平化层次**: 减少嵌套层级,使用标签页和折叠面板分层展示内容
3. **快速访问**: 高频操作(开始/停止)放在显著位置
4. **渐进呈现**: 非关键信息使用折叠面板或工具提示,减少初始视觉负担

### 关键改进

1. **顶部工具栏**: 将主要控制(开始/停止)置于顶部,易于访问
2. **卡片分组**: 使用卡片将功能分组(音频源、转录控制、结果显示、高级设置)
3. **折叠面板**: 高级设置默认折叠,减少初始视觉复杂度
4. **响应式布局**: 支持窗口大小调整,自动适配不同屏幕尺寸
5. **视觉美化**: 使用图标、颜色、圆角、阴影等现代UI元素

## Goals

1. **提升易用性**: 让新用户能够快速理解和使用界面
2. **提高效率**: 减少高频操作的点击次数
3. **改善视觉**: 打造现代、专业、美观的界面
4. **增强适配性**: 支持不同屏幕尺寸,避免显示不完整
5. **保持兼容**: 不影响现有功能和配置系统

## Alternatives Considered

### 方案1: 标签页布局
使用标签页将"控制"、"监控"、"设置"分离到不同标签页。

**优点**: 最大限度减少视觉复杂度
**缺点**: 需要切换标签页查看不同信息,不适合实时监控场景

### 方案2: 侧边抽屉式布局
使用可隐藏的侧边抽屉存放高级设置。

**优点**: 更现代的交互方式
**缺点**: 在Windows桌面应用中不够符合用户习惯

### 选择方案: 卡片+折叠面板
既保持信息的可见性,又减少初始复杂度,符合桌面应用习惯。
