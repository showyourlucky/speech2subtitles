# 文件队列转录功能实现

**实施日期**: 2025-11-06
**功能版本**: Phase 2.1
**状态**: ✅ 完成

## 📋 功能概述

为 GUI 添加了两个重要的文件管理功能：

1. **多文件队列转录** - 支持批量文件自动连续转录
2. **文件列表管理** - 支持从文件列表中移除指定文件

## 🎯 实现的功能

### 功能 1: 多文件队列转录

#### 功能描述
- 用户可以一次性选择多个音频/视频文件
- 系统自动按顺序依次转录每个文件
- 每个文件转录完成后自动切换到下一个文件
- 全部文件转录完成后显示总结信息

#### 技术实现

**文件**: `src/gui/main_window.py`

**新增属性**:
```python
# 文件转录队列管理
self.file_queue: list[str] = []  # 待转录文件队列
self.current_file_index: int = -1  # 当前转录文件索引
self.is_batch_mode: bool = False  # 是否处于批量文件转录模式
```

**新增方法**:

1. **`_start_file_transcription(file_path: str)`** (440-506行)
   - 开始单个文件的转录
   - 显示进度信息（当前文件/总文件数）
   - 创建并启动 Pipeline

2. **`_process_next_file()`** (928-985行)
   - 处理队列中的下一个文件
   - 自动切换到下一个文件
   - 全部完成时显示总结消息

3. **`_reset_file_queue()`** (508-513行)
   - 重置文件队列状态
   - 清空队列和相关标志

**修改的方法**:

1. **`_on_start_transcription()`** (342-431行)
   - 检测文件模式时初始化文件队列
   - 验证所有文件有效性
   - 启动第一个文件的转录

2. **`_on_transcription_completed()`** (874-926行)
   - 保存当前文件的历史记录
   - 检查是否有下一个文件
   - 自动触发下一个文件的转录

3. **`_on_stop_transcription()`** (516-554行)
   - 停止时清理文件队列
   - 中断批量转录流程

#### 用户体验

**启动批量转录**:

2. 点击"选择文件..."选择多个文件
3. 点击"开始转录"

**转录过程**:
- 状态栏显示: `[1/5] 正在转录: file1.mp3`


## 📊 技术要点

### 1. 文件队列管理

**队列初始化**:
```python
# 初始化文件队列
self.file_queue = file_paths.copy()
self.current_file_index = 0
self.is_batch_mode = len(file_paths) > 1
```

**队列处理流程**:
```
开始转录 -> 第一个文件
    ↓
转录完成 -> 保存历史记录 -> 检查队列
    ↓
有下一个? -> 是 -> 清理Pipeline -> 开始下一个文件
    ↓
    否 -> 显示完成消息 -> 清理队列
```

### 2. Pipeline 生命周期管理

**单文件转录**:
- 创建 Pipeline
- 启动转录
- 转录完成
- 停止 Pipeline

**多文件转录**:
- 创建 Pipeline (第一个文件)
- 启动转录
- 转录完成 -> 停止 Pipeline -> 创建新 Pipeline (第二个文件)
- 重复直到所有文件完成

### 3. UI 状态同步

**进度显示**:
```python
if self.is_batch_mode:
    progress = f"[{self.current_file_index + 1}/{len(self.file_queue)}]"
    self.status_bar.showMessage(f"{progress} 正在转录: {file_name}")
```

**按钮状态**:
- 转录中：禁用所有音频源选择
- 完成后：启用音频源选择
- 文件选中：启用"移除选中"按钮

### 4. 错误处理

**文件验证**:
```python
# 验证所有文件存在
invalid_files = [f for f in file_paths if not Path(f).exists()]
if invalid_files:
    QMessageBox.warning(...)
    return
```

**转录错误处理**:
- 单个文件错误：显示错误信息，清理队列
- 历史记录保存失败：记录错误，继续处理队列
- Pipeline 错误：清理资源，重置状态

## 📝 已知限制

### 当前限制
1. **文件格式验证**: 依赖于文件扩展名，不验证实际内容
2. **单线程处理**: 一次只能处理一个文件，不支持并行转录
3. **进度显示**: 只显示文件级别进度，不显示单个文件内的进度
4. **队列持久化**: 队列不会在程序重启后保存

### 未来改进
1. **文件格式深度验证**: 检查文件头和编码
2. **并行处理**: 多文件同时转录（需要多个 Pipeline 实例）
3. **详细进度**: 显示单个文件的转录进度百分比
4. **队列管理**: 支持队列的保存和恢复
5. **优先级控制**: 支持调整文件转录顺序
